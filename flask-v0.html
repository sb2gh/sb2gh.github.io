Website creation with Flask framework

Contents

Environment set-up¶

First create a folder where we are going to have all our files for the application

bm@Work ~ -  $ mkdir flsk
bm@Work ~ -  $ cd flsk

We then create a virtualenv that will contains all the necessary packages for flask.

bm@Work ~/flsk -  $ mkvirtualenv venv-flask
bm@Work ~/flsk -  $ source ~/.virtualenvs/venv-flask/bin/activate

Make sure that PYTHONPATH is empty, otherwise the new python will use global module as well.

export PYTHONPATH=''

We can now install flask

(venv-flask) bm@Work ~/flsk -  $ pip install flask

Once flask is installed, when we enter pip freeze we should see something like this: 

click==6.6
Flask==0.11.1
itsdangerous==0.24
Jinja2==2.8
MarkupSafe==0.23
Werkzeug==0.11.10


Hello World!¶

We create the basic file app.py 

from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
    return '<h1>Hello World!</h1>'


if __name__ == "__main__":
    app.run(debug=True)

We can now launch our app: python app.py
We can see our website at: localhost:5000

Notes:
1. The default port is 5000
2. In order to have the app seen outside the local computer we change app.run(debug=True) by app.run(host='0.0.0.0', port=5000, debug=True). If you are testing on another computer on the same network, in order to see the ip to put in the address bar, you can run ifconfig


Template¶

HTML templates should be created in order to dissociate models from views. Flask will recognize these templates. We just need to create a folder named templates at the root of our app folder.
We can then create for instance an index.html such as:

<!DOCTYPE html>
<html>
<head>
    <title>v0</title>
</head>
<body>
    <h1>Hello</h1>
    <p> paragraph</p>
    <p>Let's go to product details <a href="{{url_for('product', id=0)}}">here</a></p>
</body>
</html>

Note:
1. we have included a link in the body by using the function url_for. It will redirect to the page product.html/0.
2. We need to modify the include statement at the beginning to from flask import Flask, render_template, url_for


Forms¶

Form can be implemented in regular html but it is much easier to use the package flask-wtf. To install: pip install flask-wtf. Installing this will also help avoiding CSRF (cross-site-request-forgery) attacks.
We need to update our included packages. We will add the following includes (if we plan for example to use a radiofield and a submitfield):

from flask_wtf import Form
from wtforms.fields import RadioField, SubmitField


Main Flask decorators¶

  1. route: registers function to handle routes
  2. before_request: registers a function to run prior any request handlers
  3. before_first_request: only once at the start
  4. after_request: ...
  5. teardown_request: after request handlers - even in case of exception thrown
  6. errorhandler: custom error handler

Helper functions¶

  1. url_for(): generates links to route or static files
  2. render_template(): renders jinja2 templates
  3. redirect(): generates a redirect response
  4. jsonify(): generates a json response
  5. abort(): generates an error response
  6. flash(): registers a message to display


Large project¶

In large project it is easier to have a similar to the following tree structure (see Miguel Greenberg):

├── app/
├── config.py
├── manage.py
├── requirements.txt
└── tests/

├── __init__.py
├── emails.py

Appendix¶

Problem reload server¶

Sometimes there are sockets still running, so we need to kill the server before relaunch. Assuming the server runs on port 5000:
lsof -n -i4TCP:5000 | grep LISTEN
Get the process id (PID) and kill it, then relaunch:
kill -9 pidnumber
